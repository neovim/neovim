1. Common functionality					*clientserver*

Nvim's |RPC| functionality allows clients to programmatically control Nvim. Nvim
itself takes command-line arguments that cause it to become a client to another
Nvim running as a server.

The following command line arguments are available:

    argument			meaning	~

   --remote [+{cmd}] {file} ...					*--remote*
				Open the files in a remote Nvim server and
				block until all opened buffers are unloaded
				(closed) in the server. Exits with code 0.
				If no server is found, a warning is printed
				and files are edited locally.
				Arguments starting with + and any |-c|
				commands are treated as Ex commands and
				executed on the server after opening the
				files. Their output is written to stdout.
				All other arguments are treated as filenames.
				Use |--| before filenames to force literal
				handling for names that start with "+" or "-c".
				You cannot edit stdin this way |--|.
				Use |-p| to open each file in a new tabpage: >
				  nvim -p --server {addr} --remote {file} ...
<				Use |-Es| to suppress the warning when no
				server is found: >
				  nvim -Es --server {addr} --remote {file} ...
<			Use |+async| to open without waiting: >
			  nvim --server {addr} --remote +async {file} ...
<								*+async*
   +async		Special +{cmd} token for |--remote|. When present,
			the client opens the files on the server and exits
			immediately without waiting for the buffers to be
			closed. Can be combined with other +{cmd} arguments
			and |-p|: >
			  nvim --server {addr} --remote +async file.txt
			  nvim -p --server {addr} --remote +async file.txt
			  nvim --server {addr} --remote +async "+set ft=markdown" file.txt
<								*--remote-ui*
   --remote-ui			Display the UI of the server in the terminal.
				Fully interactive: keyboard and mouse input
				are forwarded to the server.
								*--server*
   --server {addr}		Connect to the named pipe or socket at the
				given address for executing remote commands.
				See |--listen| for specifying an address when
				starting a server.

Examples ~

Start an Nvim server listening on a named pipe at '~/.cache/nvim/server.pipe': >
    nvim --listen ~/.cache/nvim/server.pipe

Edit "file.txt" in an Nvim server and wait for it to be closed: >
    nvim --server ~/.cache/nvim/server.pipe --remote file.txt

Open "file.txt" in a new tabpage on the server: >
    nvim -p --server ~/.cache/nvim/server.pipe --remote file.txt

Fall back silently to local editing when no server is found: >
    nvim -Es --server ~/.cache/nvim/server.pipe --remote file.txt

Send input to the server via |nvim_input()|: >
    nvim --server ~/.cache/nvim/server.pipe --remote "+call nvim_input('ifoo')"

Open a file without waiting for it to be closed: >
    nvim --server ~/.cache/nvim/server.pipe --remote +async file.txt


REMOTE EDITING

The --remote argument causes each given file to be opened with |:drop| (or
|:tab| + drop when |-p| is used) on the server. Command-line `+{cmd}` and
|-c| commands are also executed on the server; their output is written to
stdout.

2. Removed flags					*clientserver-removed*

These Vim flags are not supported by Nvim. Use the alternatives shown.

*--remote-silent*               Removed. Use |-Es| instead: >
                              nvim -Es --server {addr} --remote {file}
<
*--remote-tab*                  Removed. Use |-p| instead: >
                              nvim -p --server {addr} --remote {file}
<
*--remote-tab-silent*           Removed. Use |-Es| |-p| instead: >
                              nvim -Es -p --server {addr} --remote {file}
<
*--remote-wait*                 Removed. Use |--remote| instead.

*--remote-wait-silent*          Removed. Use |-Es| instead: >
                              nvim -Es --server {addr} --remote {file}
<
*--remote-tab-wait*             Removed. Use |-p| instead: >
                              nvim -p --server {addr} --remote {file}
<
*--remote-tab-wait-silent*      Removed. Use |-Es| |-p| instead: >
                              nvim -Es -p --server {addr} --remote {file}
<
*--remote-send*                 Removed. Use |nvim_input()| instead: >
                              nvim --server {addr} --remote "+call nvim_input('ifoo')"
<
*--remote-expr*                 Removed. Use |-Es| and |--remote| +cmd: >
                              nvim -Es --server {addr} --remote "+echo 1+1" +q
<

*--serverlist*                  Removed. Use instead: >
                              nvim -Es --server {addr} --remote "+echom serverlist()" +q
<

*--servername*                  Removed.
