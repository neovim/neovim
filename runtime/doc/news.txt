BREAKING CHANGES                                                *news-breaking*

These changes may require adaptations in your config or plugins.

API

Рђб Decoration provider has `on_range()` callback.
Рђб |nvim_get_commands()| returns `complete` as a Lua function, if it was
  defined as such.

BUILD

Рђб todo

DIAGNOSTICS

Рђб |diagnostic-signs| can no longer be configured with |:sign-define| or
  |sign_define()| (deprecated in Nvim 0.10 |deprecated-0.10|).
Рђб |vim.diagnostic.disable()| and |vim.diagnostic.is_disabled()| (deprecated in
  Nvim 0.10 |deprecated-0.10|) are removed.
Рђб The legacy signature of |vim.diagnostic.enable()| (deprecated in Nvim 0.10
  |deprecated-0.10|) is no longer supported.

EDITOR

Рђб |i_CTRL-R| inserts named/clipboard registers (A-Z,a-z,0-9+) literally, like
  pasting instead of like user input. Improves performance, avoids broken
  formatting. To get the old behavior you can use `<C-R>=@x`.
Рђб Buffer name URI scheme parsing more closely follows RFC3986, so for example
  "svn+ssh:", "ed2k:", and "iris.xpc:" are recognized as URI schemes.
Рђб On Windows, Nvim no longer searches the current directory for executables
  for running external commands; prefix a relative or absolute path if you
  want the old behavior |$NoDefaultCurrentDirectoryInExePath|.

EVENTS

Рђб |ui-messages| no longer emits the `msg_show.return_prompt`, and
  `msg_history_clear` events. The `msg_clear` event was repurposed and is now
  emitted after the screen is cleared. These events arbitrarily assumed a
  message UI that mimics the legacy message grid. Benefit: reduced UI event
  traffic and more flexibility for UIs.
  The `msg_history_show` event has an additional "prev_cmd" argument.

HIGHLIGHTS

Рђб todo

LSP

Рђб JSON "null" values in LSP messages are represented as |vim.NIL| instead of `nil`.
  Missing fields (as opposed to JSON "null") are still represented as `nil`.
Рђб The function set with |vim.lsp.log.set_format_func()| is now given all
  arguments corresponding to a log entry instead of the individual arguments.
Рђб Renamed `vim.lsp.semantic_tokens` `start()/stop()` to `enable()`.
Рђб |vim.lsp.util.convert_signature_help_to_markdown_lines()| activeParameter
  handling updated:
    Рђб Values < 0 are now treated as `nil` instead of 0.
    Рђб Values outside the range of `signatures[activeSignature].parameters`
      are now treated as `nil` instead of `#signatures[activeSignature].parameters`

LUA

Рђб Renamed `vim.diff` to `vim.text.diff`.

OPTIONS

Рђб 'shelltemp' defaults to "false".

PLUGINS

Рђб Removed "shellmenu" plugin, an old menu-based quasi-snippet plugin for shell scripts.
Рђб |tohtml| is now an "opt-in" plugin. Use |:packadd| to activate it: >
  :packadd nvim.tohtml

TREESITTER

Рђб The |vim.treesitter.query.lint()| is no longer enabled by default in the
  |ft-query-plugin|.
Рђб |treesitter-directive-offset!| can now be applied to quantified captures. It
  no longer sets `metadata[capture_id].range`; it instead sets
  `metadata[capture_id].offset`. The offset will be applied in
  |vim.treesitter.get_range()|, which should be preferred over reading
  metadata directly for retrieving node ranges.
Рђб The "all" option to |Query:iter_matches()|, which was introduced in Nvim
  0.11 to aid in transitioning to the new behavior, has been removed.

NEW FEATURES                                                    *news-features*

The following new features were added.

API

Рђб |api-contract| allows existing functions to change return-type from `void => non-void`.
Рђб |nvim_win_text_height()| can limit the lines checked when a certain
  `max_height` is reached, and returns the `end_row` and `end_vcol` for which
  `max_height` or the calculated height is reached.
Рђб |vim.secure.read()| now returns `true` for trusted directories. Previously
  it would return `nil`, thus impossible to tell if the directory was actually
  trusted.
Рђб |nvim_ui_send()| writes arbitrary data to a UI's stdout. Use this to write
  escape sequences to the terminal when Nvim is running in the |TUI|.
Рђб |nvim_echo()| can set the |ui-messages| kind with which to emit the message.
Рђб |nvim_echo()| can create |Progress| messages
Рђб |nvim_get_commands()| returns `preview` and `callback` as Lua functions if
  they were so specified in `nvim_create_user_command()`.
Рђб |nvim_open_tabpage()| can open a new |tab-page|.
Рђб |nvim_open_win()| floating windows can show a 'statusline'. Plugins can use
  `style='minimal'` or `:setlocal statusline=` to hide the statusline.
Рђб |nvim_win_set_config()| can move |floating-windows| to other tabpages.
Рђб Added experimental |nvim__exec_lua_fast()| to allow remote API clients to
  execute code while nvim is blocking for input.
Рђб |vim.secure.trust()| accepts `path` for the `allow` action.
Рђб |nvim_get_chan_info()| includes `exitcode` field for terminal buffers

BUILD

Рђб A Zig-based build system has been added as an alternative to CMake. It is
  currently limited in functionality, and CMake remains the recommended option
  for the time being.
Рђб Nvim can be built without Unibilium (terminfo implementation), in which case
  the user's terminfo database won't be loaded and only internal definitions
  for the most common terminals are used: >

  make distclean && make CMAKE_EXTRA_FLAGS="-DENABLE_UNIBILIUM=0" DEPS_CMAKE_FLAGS"-DUSE_BUNDLED_UNIBILIUM=0"
<
Рђб On Windows, `tee.exe` is installed with `nvim.exe` by default so that
  commands like :make, :grep work out of the box.

DEFAULTS

Рђб 'diffopt' default value now includes "indent-heuristic" and "inline:char".
Рђб 'statusline' default is exposed as a statusline expression (previously it
  was implemented as an internal C routine).
Рђб 'statusline' includes |vim.diagnostic.status()|
Рђб 'statusline' shows exit code of finished terminal buffers
Рђб Project-local configuration ('exrc') is also loaded from parent directories.
  Unset 'exrc' to stop further search.
Рђб Mappings:
  Рђб |grt| in Normal mode maps to |vim.lsp.buf.type_definition()|
  Рђб |grx| in Normal mode maps to |vim.lsp.codelens.run()|
Рђб 'shada' default now excludes "/tmp/" and "/private/" paths to reduce clutter in |:oldfiles|.
Рђб Enabled treesitter highlighting for Markdown files

DIAGNOSTICS

Рђб |vim.diagnostic.setloclist()| and |vim.diagnostic.setqflist()| now support a
  `format` function to modify (or filter) diagnostics before being set in the
  location/quickfix list.
Рђб |vim.diagnostic.get()| now accepts an `enabled` filter to only return
  enabled or disabled diagnostics.
Рђб |vim.diagnostic.status()| returns a formatted string with current buffer
  diagnostics
Рђб |vim.diagnostic.fromqflist()| now accepts an `opts` table with
  `merge_lines` to merge multi-line compiler messages.

EDITOR

Рђб |:iput| works like |:put| but adjusts indent.
Рђб |:retab| accepts new optional parameter -indentonly to only change leading
  whitespace in indented lines.
Рђб |:uniq| deduplicates text in the current buffer.
Рђб |omnicompletion| in `help` buffer. |ft-help-omni|
Рђб |:help!| has DWIM ("Do What I Mean") behavior: it tries to guess the help
  tag at cursor. In help buffers, 'keywordprg' defaults to ":help!". For
  example, try "K" anywhere in this code: >
    local v = vim.version.parse(vim.system({'foo'}):wait().stdout)
Рђб Setting "'0" in 'shada' prevents storing the jumplist in the shada file.
Рђб 'shada' now correctly respects "/0" and "f0".
Рђб |prompt-buffer| supports multiline input/paste, undo/redo, and o/O normal
  commands.
Рђб 'wildchar' now enables completion in search contexts using |/|, |?|, |:g|, |:v|
   and |:vimgrep| commands.
Рђб For security, 'exrc' no longer shows "(a)llow". Instead you must "(v)iew",
  then run `:trust` (or `:trust [file]`, but that has a TOCTOU risk).
Рђб |gx| in help buffers opens the online documentation for the tag under the
  cursor.
Рђб |:source| with a range in non-Lua files (e.g., vimdoc) now detects Lua
  codeblocks via treesitter and executes them as Lua instead of Vimscript.
Рђб |:Undotree| for visually navigating the |undo-tree|
Рђб |:wall| permits a |++p| option for creating parent directories when writing
  changed buffers.
Рђб The |:DiffTool| command compares directories (and files).

EVENTS

Рђб A new `empty` message kind is emitted for an empty (e.g. `:echo ""`) message.
Рђб |CmdlineLeave| sets |v:char| to the character that stops the Cmdline mode.
Рђб |CmdlineLeavePre| triggered before preparing to leave the command line.
Рђб New `append`, `id` and `trigger` parameter for |ui-messages| `msg_show` event.
Рђб 'rulerformat' is emitted as `msg_ruler` when not part of the statusline.
Рђб Creating or updating a progress message with |nvim_echo()| triggers a |Progress| event.
Рђб |MarkSet| is triggered after a |mark| is set by the user (currently doesn't
  support implicit marks like |'[| or |'<|, Рђд).
Рђб |SessionLoadPre| is triggered before loading a |Session| file.
Рђб |TabClosedPre| is triggered before closing a |tabpage|.
Рђб New `terminator` parameter for |TermRequest| event.

HIGHLIGHTS

Рђб |hl-DiffTextAdd| highlights added text within a changed line.
Рђб |hl-OkMsg| |hl-StderrMsg| |hl-StdoutMsg|
Рђб |hl-SnippetTabstopActive| highlights the currently active tabstop.
Рђб |hl-PmenuBorder| |hl-PmenuShadow| |hl-PmenuShadowThrough| see 'pumborder'.
Рђб |nvim_set_hl()| and |nvim_get_hl()| support the SGR attributes "dim",
  "blink", "conceal", and "overline".

LSP

Рђб |:lsp| can be used to interactively manage LSP clients.
Рђб |vim.lsp.ClientConfig| gained `workspace_required`.
Рђб You can control the priority of |vim.lsp.Config| `root_markers`.
Рђб Support for `textDocument/documentColor`: |lsp-document_color|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_documentColor
Рђб Support for `textDocument/colorPresentation |lsp-document_color|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_colorPresentation
Рђб The `textDocument/diagnostic` request now includes the previous id in its
  parameters.
Рђб |vim.lsp.enable()| start/stops clients as necessary and detaches
  non-applicable LSP clients.
Рђб |vim.lsp.is_enabled()| checks if a LSP config is enabled (without
  "resolving" it).
Рђб |vim.lsp.get_configs()| gets all LSP configs matching an optional filter.
Рђб Support for `workspace/diagnostic`: |vim.lsp.buf.workspace_diagnostics()|
  https://microsoft.github.io/language-server-protocol/specifications/specification-current/#workspace_dagnostics
Рђб Incremental selection is now supported via `textDocument/selectionRange`.
  `an` selects outwards and `in` selects inwards.
Рђб Support for multiline semantic tokens.
Рђб Support for the `disabled` field on code actions.
Рђб The function form of `cmd` in a vim.lsp.Config or vim.lsp.ClientConfig
  receives the resolved config as the second arg: `cmd(dispatchers, config)`.
Рђб Support for annotated text edits.
Рђб `:checkhealth vim.lsp` is now available to check which buffers the active LSP features are attached to.
Рђб LSP `DiagnosticRelatedInformation` is now shown in
  |vim.diagnostic.open_float()|. It is read from the LSP diagnostic object
  stored in the `user_data` field.
Рђб When inside the float created by |vim.diagnostic.open_float()| and the
  cursor is on a line with `DiagnosticRelatedInformation`, |gf| can be used to
  jump to the problematic location.
Рђб Support for `textDocument/linkedEditingRange`: |lsp-linked_editing_range|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_linkedEditingRange
Рђб Support for related documents in pull diagnostics:
  https://microsoft.github.io/language-server-protocol/specifications/specification-current/#relatedFullDocumentDiagnosticReport
Рђб |vim.lsp.buf.signature_help()| supports "noActiveParameterSupport".
Рђб Support for `textDocument/inlineCompletion` |lsp-inline_completion|
  https://microsoft.github.io/language-server-protocol/specifications/lsp/3.18/specification/#textDocument_inlineCompletion
  See |lsp-inline_completion| for quickstart instructions.
Рђб Support for `textDocument/onTypeFormatting`: |lsp-on_type_formatting|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_onTypeFormatting
Рђб The filter option of |vim.lsp.buf.code_action()| now receives the client ID as an argument.
Рђб |vim.lsp.ClientConfig| `exit_timeout` decides the time waited before "stop"
  escalates to "force-stop" for |vim.lsp.enable()|, |Client:stop()|, and
  during Nvim shutdown.
  Рђб `exit_timeout` graduated from "experimental" `flags.exit_timeout`
    to a top-level field. Defaults to `false`.
Рђб |Client:stop()| accepts `force` as an integer, which is treated as the time
  to wait before before stop escalates to force-stop.
Рђб |lsp-completion|:
  Рђб |vim.lsp.completion.enable()| gained a `cmp` option for custom ordering.
  Рђб Supports colored symbol preview for color items.
  Рђб Shows a preview ("completionItem/resolve") if 'completeopt' has "popup".
Рђб Support for `workspace/diagnostic/refresh`:
  https://microsoft.github.io/language-server-protocol/specification/#diagnostic_refresh
Рђб Support for dynamic registration for `textDocument/diagnostic`
Рђб |vim.lsp.buf.rename()| now highlights the symbol being renamed using the

Рђб Support for `textDocument/semanticTokens/range`.
Рђб Support for `textDocument/codeLens` |lsp-codelens| has been reimplemented:
  https://microsoft.github.io/language-server-protocol/specifications/lsp/3.18/specification/#textDocument_codeLens
Рђб Code lenses now display as virtual lines

PERFORMANCE

Рђб |vim.glob.to_lpeg()| uses a new LPeg-based implementation (Peglob) that
  provides ~50% speedup for complex patterns. The implementation restores
  support for nested braces and follows LSP 3.17 specification with
  additional constraints for improved correctness and resistance to
  backtracking edge cases.
Рђб |i_CTRL-R| inserts named/clipboard registers literally, 10x speedup.
Рђб LSP `textDocument/semanticTokens/range` is supported, which requests tokens
  for the viewport (visible screen) only.
Рђб |:packadd| doesn't invalidate the cached Lua package path. Instead the cache
  gets updated in place. This might make a big startuptime difference for
  certain |init.lua| patterns where multiple |:packadd| or |vim.pack.add()|
  calls are interspersed with other code.

TERMINAL

Рђб CSI 3 J (the sequence to clear terminal scrollback) is now supported.
Рђб DEC private mode 2026 (synchronized output) is now supported. Applications

Рђб A suspended PTY process is now indicated by "[Process suspended]" at the
  bottom-left of the buffer and can be resumed by pressing a key.
Рђб On terminal exit, "[Process exited]" is shown as virtual text (instead of
  modifying buffer contents), and exit code is shown in statusline.
  Рђб You can disable "[Process exited]" entirely, see |terminal-config|.

  Рђб Avoids "Press ENTER" interruptions.

Рђб 'busy' status is shown in default statusline with symbol РЌљ

CHANGED FEATURES                                                 *news-changed*

These existing features changed their behavior.

Рђб 'pumblend' does not apply special attributes (bold, underline) from the
  background layer to the foreground layer.
Рђб |gv| works in operator pending mode and does not abort.
Рђб 'smartcase' applies to completion filtering.
Рђб 'spellfile' location defaults to `stdpath("data").."/site/spell/"` instead of
  the first writable directory in 'runtimepath'.
Рђб |vim.version.range()| doesn't exclude `to` if it is equal to `from`.
Рђб |$VIM| and |$VIMRUNTIME| no longer check for Vim version-specific runtime
  directory `vim{number}` (e.g. `vim82`).
Рђб 'scrollback' maximum value increased from 100000 to 1000000
Рђб |matchfuzzy()| and |matchfuzzypos()| use an improved fuzzy matching algorithm
  (same as fzy).
Рђб Windows: Paths like "\Windows" and "/Windows" are now considered to be
  absolute paths (to the current drive) and no longer relative.
Рђб When 'shelltemp' is off, shell commands now use `pipe()` and not `socketpair()`
  for input and output. This matters mostly for Linux where some command lines
  using "/dev/stdin" and similar would break as these special files can be
  reopened when backed by pipes but not when backed by socket pairs.
