BREAKING CHANGES                                                *news-breaking*

These changes may require adaptations in your config or plugins.

API

‚ÄĘ |nvim_create_autocmd()|, |nvim_exec_autocmds()| and |nvim_clear_autocmds()|
  no longer treat an empty non-nil pattern as nil.
‚ÄĘ |nvim_clear_autocmds()| no longer treats an empty array event as nil.
‚ÄĘ |vim.o|, |vim.opt|, and |nvim_set_option_value()| expand `~` and environment
  variables (|expand-env|).

DIAGNOSTICS

‚ÄĘ `vim.diagnostic.Opts.Status.format` no longer accepts the table mapping
  from severity to text. Use `vim.diagnostic.Opts.Signs.text` instead.
‚ÄĘ `current_line` property of |vim.diagnostic.Opts.VirtualLines| and
  |vim.diagnostic.Opts.VirtualText| is now applied on |CursorHold| event.
  Make sure your 'updatetime' is reasonable.

EDITOR

‚ÄĘ On Windows, the |trust| db now stores paths with "/" slashes. This means the
  trust store will be reset (one time).
‚ÄĘ `stdpath("log")` moved to `stdpath("state")/logs`.
‚ÄĘ |cmdwin| (|q:|, |q/|, |q?|, |c_CTRL-F|):
  ‚ÄĘ Only supports one "cmdline level", not "infinite recursion".
  ‚ÄĘ Expr-register (|i_CTRL-R_=|, |c_CTRL-R_=|) no longer supports opening
    cmdwin via |c_CTRL-F|.
‚ÄĘ Behavior of |:restart| changed. Use "!" (|:restart!|) to get the old behavior.
  ‚ÄĘ |ZR| now performs |:restart|. Add a count to change the behavior.

EVENTS

‚ÄĘ |BufModifiedSet| has been removed. Use the |OptionSet| event with pattern
  "modified" instead.
‚ÄĘ |ui-messages| `msg_show.bufwrite` and `msg_show.completion` messages are now
  `msg_show.progress` events.

LSP

‚ÄĘ `vim.lsp.ClientConfig.cmd` given as a string array now uses
  `vim.lsp.ClientConfig.root_dir` as its default process working directory.
‚ÄĘ `client.attached_buffers[buf]` now stores `languageId` string (was boolean).

LUA

‚ÄĘ "standalone" Lua interpreter mode `nvim -ll` was removed. Use |-l| script
  mode instead.
‚ÄĘ |vim.opt| no longer supports chaining multiple infix operators (e.g.
  `vim.opt.wildignore + '*.o' + '*.obj'`). Instead, use tables:
  `vim.opt.wildignore + {'*.o', '*.obj'}`

OPTIONS

‚ÄĘ The 'statusline' `%=` item is no longer ignored in item groups. It applies
  to the containing item group and its `minwid`.
  An `%<` item inside an item group now only applies to the containing group
  and its `maxwid`. Therefore, multiple `%<` can now be effective.
  The default truncation of item groups is now at the first contained item.
  To restore the previous truncation from the left, add a `%<` to the start.
  The same applies to 'rulerformat', 'statuscolumn', 'tabline', 'winbar',
  'titlestring', and 'iconstring'.

‚ÄĘ Removed: ctxget(), ctxpop(), ctxpush(), ctxset(), ctxsize(). Use

NEW FEATURES                                                    *news-features*

The following new features were added.

API

‚ÄĘ |nvim_win_resize()| resizes a window with a specified anchor edge.
  |nvim_win_set_height()| and |nvim_win_set_width()| are now deprecated.
‚ÄĘ |api-contract| treats `opts` and any parameters thereafter, as optional
  (equivalent to passing an empty dict). Clients may omit such optional
  parameters.
  ‚ÄĘ The Lua |vim.api| bridge is also a "client" and thus may omit `opts` and
    any following parameters: >lua
    -- Before:
    vim.print(vim.api.nvim_get_mark('A', {}))
    -- After:
    vim.print(vim.api.nvim_get_mark('A'))
‚ÄĘ |nvim_buf_call()| and |nvim_win_call()| now preserve multiple return values.
‚ÄĘ |nvim_set_hl()| supports "font" key.
‚ÄĘ |nvim_open_win()| `zindex` controls whether the UI will use a dimmed cursor
  shape when an unfocused float is on top of the cursor.
‚ÄĘ |nvim_echo()| distinguishes zero percent from omitted percent for Progress
  events.
‚ÄĘ |nvim_create_user_command()| accepts `desc` for Vimscript commands.
‚ÄĘ Support for tabpage-local options ('cmdheight'):
  ‚ÄĘ |gettabvar()|
  ‚ÄĘ |gettabwinvar()|
  ‚ÄĘ |nvim_get_option_info2()|
  ‚ÄĘ |nvim_get_option_value()|
  ‚ÄĘ |nvim_set_option_value()|
‚ÄĘ |nvim_buf_set_extmark()| `virt_lines_overflow` accepts "wrap" to enable
  wrapping onto extra rows and "auto" which enables horizontal scrolling when
  'nowrap' is set and wrapping when 'wrap' is set.
‚ÄĘ |nvim_set_option_value()| accepts a new `operation` field to modify the
  existing option value.
‚ÄĘ |nvim_set_option_value()| returns the new option value.
‚ÄĘ |nvim_create_autocmd()| is now |api-fast|, so it can be called from a fast
  event context (e.g. |vim.uv| callbacks).

BUILD

‚ÄĘ Building using "zig build" requires zig 0.16.x.
‚ÄĘ zig build: "-Dcross" option was removed. Often cross-compilation is now
  detected, so e.g. from a linux host: >
   zig build -Dtarget=aarch64-macos
<  will automatically compile a host Lua for use during build.
‚ÄĘ zig build: The new "-Dhost={target_string}" option can be used to override
  the used host. Use "-Dhost=native" to force cross-compiling or "-Dhost="
  (empty string) to assume that target binaries can run on the host during the
  build process (e.g. if target is x86 on a x86_64 system, or if emulation set
  up with binfmt or similar).

DIAGNOSTICS

‚ÄĘ |vim.diagnostic.status()| uses the severity names defined by the `signs`
  field of |vim.diagnostic.config()|, if any.
‚ÄĘ `virtual_lines.overflow` in |vim.diagnostic.config()| controls
  how virtual lines wider than the window are displayed.

EDITOR

‚ÄĘ |:command-completion-customlist| can return a list of dictionaries with
  kind/menu/info/abbr for the popup menu.
‚ÄĘ |cmdwin| (|q:|, |q/|, |q?|, |c_CTRL-F|) is implemented as a "normal"
  buffer+window instead of a nested-state modal loop:
  ‚ÄĘ You can create/navigate windows/tabpages while cmdwin is active. Chaos!
  ‚ÄĘ 'inccommand' works in cmdwin!
  ‚ÄĘ API calls (e.g. |nvim_buf_delete()|) that previously failed with
    "E11: Invalid in command-line window" while cmdwin was open, now work
    normally. Async plugins no longer need special |CmdwinLeave| workarounds.
  ‚ÄĘ The |cmdwin-char| is shown via 'statuscolumn'.
‚ÄĘ |gf| and |<cfile>| support `file://‚Ä¶` URIs.
‚ÄĘ |:log| opens log files.
‚ÄĘ |:marks| can list |extmarks| for a given |namespace|.
‚ÄĘ |:marks| gained tab-completion.
‚ÄĘ |:restart| saves/restores the current session (window layout, buffers, ‚Ä¶).
  ‚ÄĘ |:restart!| (with a bang "!") does not save/restore the session.
‚ÄĘ |ZR| restarts Nvim (|:restart|).
‚ÄĘ |:detach| with range "%" detaches all UIs except the current one.
‚ÄĘ |:uptime| displays uptime.
‚ÄĘ |:packupdate| and |:packdel| for managing |vim.pack|.
‚ÄĘ 'scrollback' is now also valid in |prompt-buffer| buffers to limit the
  number of history lines kept above the prompt.
‚ÄĘ |v_al| and |v_il| text objects select the whole buffer and the current line
  without leading or trailing white space.
‚ÄĘ 'autoread' uses file system watchers to detect external changes in
  real-time, instead of only on |FocusGained| or |:checktime|.
‚ÄĘ During |complete()|-triggered completion, CTRL-N and CTRL-P are now subject
  to insert-mode mappings.
‚ÄĘ Multi-byte characters, translated by 'langmap', now invoke correct
  mappings.
  Example: >vim
    set langmap=√Ķ]
<    Typing √Ķ√Ķ in |Normal-mode| now invokes a mapping bound to ]].
‚ÄĘ New |complete-items| field "commit_chars": characters that accept the
  selected completion item when typed.

EVENTS

‚ÄĘ |:delmarks| now triggers the |MarkSet| autocommand with line==col==0, same
  as |nvim_buf_del_mark()|
‚ÄĘ |ChanClose| is triggered after a channel is closed, before it is removed.
‚ÄĘ |SessionWritePre| event emits just before |:mksession|.
‚ÄĘ |TextPutPre| and |TextPutPost| are triggered before/after putting text.
‚ÄĘ |TabMoved| is triggered when tabs are reordered.

HIGHLIGHTS

‚ÄĘ `Dimmed` for text that should be de-emphasized.
‚ÄĘ |vim.hl.hl_op()| highlights text regions for |TextYankPost| and
  |TextPutPost| events. It replaces the now deprecated `vim.hl.on_yank()`.

LSP

‚ÄĘ LSP capabilities:
  ‚ÄĘ Completion supports `CompletionItem.preselect` if 'completeopt' has
    "preselect". https://microsoft.github.io/language-server-protocol/specification/#completionClientCapabilities
  ‚ÄĘ `textDocument/foldingRange` |vim.lsp.foldtext()| highlights collapsed text.
    https://microsoft.github.io/language-server-protocol/specification/#textDocument_foldingRange
  ‚ÄĘ Completon supports `CompletionList.applyKind`: a server can ask for
    `commitCharacters` and `data` from `itemDefaults` to be merged with the
    item's own values instead of replacing them.
    https://microsoft.github.io/language-server-protocol/specification/#completionItemApplyKinds
‚ÄĘ `:checkhealth vim.lsp` highlights the "current buffer".
‚ÄĘ |vim.lsp.buf.declaration()|, |vim.lsp.buf.definition()|,
  |vim.lsp.buf.type_definition()|, and |vim.lsp.buf.implementation()|
  now follows 'switchbuf'.
‚ÄĘ |vim.lsp.buf.declaration()|, |vim.lsp.buf.definition()|,
  |vim.lsp.buf.type_definition()|, and |vim.lsp.buf.implementation()|
  now supports passing arbitrary positions.
‚ÄĘ Support for nested snippets.
‚ÄĘ |lsp-completion| supports `commitCharacters`: typing a commit character
  while a completion item is selected accepts the item as with
  |complete_CTRL-Y|, then inserts the character. Previously
  `commitCharacters` had no effect.

LUA

‚ÄĘ |vim.ui.img| can display images. Use `:checkhealth vim.health` to confirm
  your terminal supports it.
‚ÄĘ |vim.net.request()| can specify custom headers by passing `opts.headers`.
‚ÄĘ |vim.net.request()| can now accept  `method` param overload for multiple HTTP methods.
‚ÄĘ |writefile()| treats Lua and RPC strings as |Blob|, so it can be used to
  write binary data.
‚ÄĘ |vim.fs.dir()| with `opts.err=true`, reports errors. An inaccessible root
  dir yields a single (name, nil, err) item.
‚ÄĘ |vim.fs.find()| returns a list of errors as its second return value.
‚ÄĘ |vim.fs.mkdir()| creates directories, including parent directories with
  `opts.parents=true`.
‚ÄĘ |vim.filetype.inspect()| returns a copy of the internal tables used for
  filetype detection.
‚ÄĘ Added `__eq` metamethod to |vim.VersionRange|. 2 distinct but representing
  the same range instances now compare equal.
‚ÄĘ Documentation for |vim.fn| now points to preferred Lua alternatives when
  available.
‚ÄĘ |vim.list.unique()| and |vim.list.bisect()| now support passing a string
  as a shorthand of a `key`
‚ÄĘ |vim.isnil()| tests if a value is `nil` or |vim.NIL|.
‚ÄĘ |vim.nonnil()| returns the first argument which is not nil.
‚ÄĘ |vim.npcall()| calls the function `fn` in protected-mode like |pcall()|,
  but returns `nil` on error.
‚ÄĘ |vim.pos| can now convert between positions and buffer offsets.
‚ÄĘ |vim.pos| and |vim.range| can now convert between mark positions.
‚ÄĘ |vim.ui.input()| now allows setting input scope.
‚ÄĘ |vim.log| provides a logging interface.
‚ÄĘ |vim.pack.get()| output includes revision of a pending update.
‚ÄĘ |vim.pack.get()| can fetch new updates before computing the output.
‚ÄĘ |vim.o| now accepts table style values for assignment.
‚ÄĘ |vim.keycode()| returns structured info as return value 2.
‚ÄĘ |Iter:count()| counts items in the iterator.

OPTIONS

‚ÄĘ 'ttyfast' can be disabled during startup by setting |$NVIM_NOTTYFAST|.
‚ÄĘ 'scrolloffpad' allows vertically centering cursor at the end of file.
‚ÄĘ 'shortmess' flag |shm-u| silences undo/redo messages.
‚ÄĘ 'statusline' supports |stl-%0{| to insert the expression result verbatim.
  The `%=` item is supported within item groups to separate alignment sections
  within the containing group until its `minwid` is reached. The truncation
  of an item group to its `maxwid` can be controlled with the `%<` item.
  The same applies to 'rulerformat', 'statuscolumn', 'tabline', 'winbar',
  'titlestring', and 'iconstring'.
‚ÄĘ 'winpinned' prevents window from closing unless specifically targeted.
‚ÄĘ 'packlockfile' sets the path used for |vim.pack-lockfile|.
‚ÄĘ 'previewpopup' decides how the preview window is displayed.

PERFORMANCE

‚ÄĘ |treesitter-highlight| performance on large injection-heavy files improves
  by 50% to 100% by reusing edited child-tree ranges.
‚ÄĘ Nvim architecture allows pure-Lua implementations of some `vim.fn`
  functions, which skips the Vimscript <=> Lua "bridge" (no data
  conversion/marshalling) entirely, if the `vim.fn` function is called from
  Lua.
‚ÄĘ Internal LSP data is cleared using `table.clear`, which reduces GC and
  memory reallocation during each data reset.
‚ÄĘ RPC client avoids string allocations when parsing Content-Length messages.
‚ÄĘ LSP: "overscan" semantic_token range requests to avoid flicker.

CHANGED FEATURES                                                 *news-changed*

These existing features changed their behavior.

‚ÄĘ `nvim_exec_autocmds({buf=‚Ä¶})` runs in the context of the target buffer.
‚ÄĘ |OptionSet| is no longer triggered during startup by automatic
  |'background'| detection.
‚ÄĘ Editing a local directory now shows its contents in a `filetype=directory`
  buffer. See |dir|.
‚ÄĘ |:Open| with no arguments uses the current file.
‚ÄĘ The "buffer" key was renamed to "buf" in these functions (but the old name
  "buffer" is still accepted, for backwards compatibility):
  ‚ÄĘ |nvim_clear_autocmds()|
  ‚ÄĘ |nvim_create_autocmd()|
  ‚ÄĘ |nvim_exec_autocmds()|
  ‚ÄĘ |nvim_get_autocmds()|
‚ÄĘ |vim.keymap.set()| and |vim.keymap.del()| accept a list of strings for `lhs`.
‚ÄĘ 'statuscolumn' click function labels can be different across rows.
  |v:virtnum| and |v:lnum| are available during the click function callback.
‚ÄĘ Markdown inline highlighting now conceals the backslash in backslash escapes.
‚ÄĘ Markdown inline backslash escapes and hard line breaks no longer use the
  `@string.escape` capture.
‚ÄĘ |vim.lsp.util.open_floating_preview()| windows (e.g. |vim.lsp.buf.hover()|)
  converted to normal windows (e.g. with |CTRL-W_H|) are no longer closed
  automatically.

REMOVED FEATURES                                                 *news-removed*

These deprecated features were removed.

‚ÄĘ Vimscript functions ctxget(), ctxpop(), ctxpush(), ctxset(), ctxsize()
  and the "context stack". Use |nvim_get_context()| and |nvim_load_context()|
  with a script-owned list instead.
‚ÄĘ |vim.lsp.buf.declaration()|, |vim.lsp.buf.definition()|,
  |vim.lsp.buf.type_definition()|, and |vim.lsp.buf.implementation()|
  no longer accept the `reuse_win` option.
