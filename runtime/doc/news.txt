BREAKING CHANGES                                                *news-breaking*

These changes may require adaptations in your config or plugins.

DIAGNOSTICS

‚ÄĘ |diagnostic-signs| can no longer be configured with |:sign-define| or
  |sign_define()| (deprecated in Nvim 0.10 |deprecated-0.10|).
‚ÄĘ |vim.diagnostic.disable()| and |vim.diagnostic.is_disabled()| (deprecated in
  Nvim 0.10 |deprecated-0.10|) are removed.
‚ÄĘ The legacy signature of |vim.diagnostic.enable()| (deprecated in Nvim 0.10
  |deprecated-0.10|) is no longer supported.

EDITOR

‚ÄĘ todo

EVENTS

‚ÄĘ |ui-messages| no longer emits the `msg_show.return_prompt`, and
  `msg_history_clear` events. The `msg_clear` event was repurposed and is now
  emitted after the screen is cleared. These events arbitrarily assumed a
  message UI that mimicks the legacy message grid. Benefit: reduced UI event
  traffic and more flexibility for UIs.
  The `msg_history_show` event has an additional "prev_cmd" argument.
‚ÄĘ A new `empty` message kind is emitted for an empty (e.g. `:echo ""`) message.

HIGHLIGHTS

‚ÄĘ todo

LSP

‚ÄĘ `root_markers` in |vim.lsp.Config| can now be ordered by priority.
‚ÄĘ The function set with |vim.lsp.log.set_format_func()| is now given all
  arguments corresponding to a log entry instead of the individual arguments.
‚ÄĘ `vim.lsp.semantic_tokens.start/stop` now renamed to
  `vim.lsp.semantic_tokens.enable`

LUA

‚ÄĘ Renamed `vim.diff` to `vim.text.diff`.
‚ÄĘ |vim.net.request()| adds a minimal HTTP GET API using curl.

OPTIONS

‚ÄĘ 'shelltemp' defaults to "false".

TREESITTER

‚ÄĘ |treesitter-directive-offset!| can now be applied to quantified captures. It
  no longer sets `metadata[capture_id].range`; it instead sets
  `metadata[capture_id].offset`. The offset will be applied in
  |vim.treesitter.get_range()|, which should be preferred over reading
  metadata directly for retrieving node ranges.

NEW FEATURES                                                    *news-features*

The following new features were added.

API

‚ÄĘ |api-contract| allows existing functions to change return-type from `void =>
  non-void`.
‚ÄĘ |nvim_win_text_height()| can limit the lines checked when a certain
  `max_height` is reached, and returns the `end_row` and `end_vcol` for which
  `max_height` or the calculated height is reached.
‚ÄĘ |vim.secure.read()| now returns `true` for trusted directories. Previously
  it would return `nil`, which made it impossible to tell if the directory was
  actually trusted.
‚ÄĘ Added |vim.lsp.is_enabled()| to check if a given LSP config has been enabled
  by |vim.lsp.enable()|.
‚ÄĘ |nvim_echo()| can set the |ui-messages| kind with which to emit the message.

BUILD

‚ÄĘ A Zig-based build system has been added as an alternative to CMake. It is
  currently limited in functionality, and CMake remains the recommended option
  for the time being.

DEFAULTS

‚ÄĘ 'statusline' default is exposed as a statusline expression (previously it
  was implemented as an internal C routine).
‚ÄĘ Project-local configuration ('exrc') is also loaded from parent directories.
  Unset 'exrc' to stop further search.
‚ÄĘ Mappings:
  ‚ÄĘ |grt| in Normal mode maps to |vim.lsp.buf.type_definition()|

DIAGNOSTICS

‚ÄĘ |vim.diagnostic.setloclist()| and |vim.diagnostic.setqflist()| now support a
  `format` function to modify (or filter) diagnostics before being set in the
  location/quickfix list.
‚ÄĘ |vim.diagnostic.get()| now accepts an `enabled` filter to only return
  enabled or disabled diagnostics.

EDITOR

‚ÄĘ |:iput| works like |:put| but adjusts indent.
‚ÄĘ |:retab| accepts new optional parameter -indentonly to only change leading
  whitespace in indented lines.
‚ÄĘ |:uniq| deduplicates text in the current buffer.
‚ÄĘ |omnicompletion| in `help` buffer. |ft-help-omni|
‚ÄĘ Setting "'0" in 'shada' prevents storing the jumplist in the shada file.
‚ÄĘ 'shada' now correctly respects "/0" and "f0".
‚ÄĘ |prompt-buffer| supports multiline input/paste, undo/redo, and o/O normal
  commands.
‚ÄĘ 'wildchar' now enables completion in search contexts using |/|, |?|, |:g|, |:v|
   and |:vimgrep| commands.

EVENTS

‚ÄĘ |CmdlineLeavePre| triggered before preparing to leave the command line.
‚ÄĘ New `append` paremeter for |ui-messages| `msg_show` event.

HIGHLIGHTS

‚ÄĘ |hl-DiffTextAdd| highlights added text within a changed line.
‚ÄĘ |hl-StderrMsg| |hl-StdoutMsg|

LSP

‚ÄĘ |vim.lsp.ClientConfig| gained `workspace_required`.
‚ÄĘ You can control priority of |vim.lsp.Config| `root_markers`.
‚ÄĘ Support for `textDocument/documentColor`: |lsp-document_color|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_documentColor
‚ÄĘ Support for `textDocument/colorPresentation |lsp-document_color|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_colorPresentation
‚ÄĘ The `textDocument/diagnostic` request now includes the previous id in its
  parameters.
‚ÄĘ |vim.lsp.enable()| start/stops clients as necessary. And detaches
  non-applicable LSP clients.
‚ÄĘ |vim.lsp.is_enabled()| checks if a LSP config is enabled (without
  "resolving" it).
‚ÄĘ Support for `workspace/diagnostic`: |vim.lsp.buf.workspace_diagnostics()|
  https://microsoft.github.io/language-server-protocol/specifications/specification-current/#workspace_dagnostics
‚ÄĘ Incremental selection is now supported via `textDocument/selectionRange`.
  `an` selects outwards and `in` selects inwards.
‚ÄĘ Support for multiline semantic tokens.
‚ÄĘ Support for the `disabled` field on code actions.
‚ÄĘ The function form of `cmd` in a vim.lsp.Config or vim.lsp.ClientConfig
  receives the resolved config as the second arg: `cmd(dispatchers, config)`.
‚ÄĘ Support for annotated text edits.
‚ÄĘ `:checkhealth vim.lsp` is now available to check which buffers the active LSP features are attached to.
‚ÄĘ LSP `DiagnosticRelatedInformation` is now shown in
  |vim.diagnostic.open_float()|. It is read from the LSP diagnostic object
  stored in the `user_data` field.
‚ÄĘ When inside the float created by |vim.diagnostic.open_float()| and the
  cursor is on a line with `DiagnosticRelatedInformation`, |gf| can be used to
  jump to the problematic location.
‚ÄĘ Support for `textDocument/linkedEditingRange`: |lsp-linked_editing_range|
  https://microsoft.github.io/language-server-protocol/specification/#textDocument_linkedEditingRange
‚ÄĘ Support for `textDocument/inlineCompletion` |lsp-inline_completion|
  https://microsoft.github.io/language-server-protocol/specifications/lsp/3.18/specification/#textDocument_inlineCompletion

LUA

‚ÄĘ Lua type annotations for `vim.uv`.
‚ÄĘ |vim.hl.range()| now allows multiple timed highlights.
‚ÄĘ |vim.tbl_extend()| and |vim.tbl_deep_extend()| now accept a function behavior argument.
‚ÄĘ |vim.fs.root()| can define "equal priority" via nested lists.
‚ÄĘ |vim.version.range()| output can be converted to human-readable string with |tostring()|.
‚ÄĘ |vim.version.intersect()| computes intersection of two version ranges.
‚ÄĘ |Iter:take()| and |Iter:skip()| now optionally accept predicates.
‚ÄĘ Built-in plugin manager |vim.pack|

OPTIONS

‚ÄĘ 'autowriteall' writes all buffers upon receiving `SIGHUP`, `SIGQUIT` or `SIGTSTP`.
‚ÄĘ 'chistory' and 'lhistory' set size of the |quickfix-stack|.
‚ÄĘ 'completefuzzycollect' enables fuzzy collection of candidates for (some)
  |ins-completion| modes.
‚ÄĘ 'complete' new flags:
  ‚ÄĘ "F{func}"   complete using given function
  ‚ÄĘ "F"         complete using 'completefunc'
  ‚ÄĘ "o"         complete using 'omnifunc'
‚ÄĘ 'complete' allows limiting matches for sources using "{flag}^<limit>".
‚ÄĘ 'completeopt' flag "nearset" sorts completion results by distance to cursor.
‚ÄĘ 'diffanchors' specifies addresses to anchor a diff.
‚ÄĘ 'diffopt' `inline:` configures diff highlighting for changes within a line.
‚ÄĘ 'grepformat' is now a |global-local| option.
‚ÄĘ 'maxsearchcount' sets maximum value for |searchcount()| and defaults to 999.
‚ÄĘ 'pummaxwidth' sets maximum width for the completion popup menu.
‚ÄĘ 'winborder' "bold" style, custom border style.
‚ÄĘ |g:clipboard| accepts a string name to force any builtin clipboard tool.
‚ÄĘ 'busy' sets a buffer "busy" status. Indicated in the default statusline.

PERFORMANCE

‚ÄĘ |vim.glob.to_lpeg()| uses a new LPeg-based implementation (Peglob) that
  provides ~50% speedup for complex patterns. The implementation restores
  support for nested braces and follows LSP 3.17 specification with
  additional constraints for improved correctness and resistance to
  backtracking edge cases.

PLUGINS

‚ÄĘ Customize :checkhealth by handling a `FileType checkhealth` event.
  |health-usage|

‚ÄĘ todo

TERMINAL

‚ÄĘ |nvim_open_term()| can be called with a non-empty buffer. The buffer
  contents are piped to the PTY and displayed as terminal output.

TREESITTER

‚ÄĘ todo

TUI

‚ÄĘ |TermResponse| now supports DA1 and APC query responses.

UI

‚ÄĘ |:restart| restarts Nvim and reattaches the current UI.
‚ÄĘ |:checkhealth| shows a summary in the header for every healthcheck.
‚ÄĘ |ui-multigrid| provides composition information and absolute coordinates.
‚ÄĘ `vim._extui` provides an experimental commandline and message UI intended to
  replace the message grid in the TUI.
‚ÄĘ Error messages are more concise:
  ‚ÄĘ "Error detected while processing:" changed to "Error in:".
  ‚ÄĘ "Error executing Lua:" changed to "Lua:".
‚ÄĘ 'busy' status is shown in default statusline with symbol ‚óź

VIMSCRIPT

‚ÄĘ |cmdcomplete_info()| gets current cmdline completion info.
‚ÄĘ |getcompletiontype()| gets command-line completion type for any string.
‚ÄĘ |prompt_getinput()| gets current user-input in prompt-buffer.

CHANGED FEATURES                                                 *news-changed*

These existing features changed their behavior.

‚ÄĘ |gv| works in operator pending mode and does not abort.
‚ÄĘ 'smartcase' applies to completion filtering.
‚ÄĘ 'spellfile' location defaults to `stdpath("data").."/site/spell/"` instead of
  the first writable directory in 'runtimepath'.
‚ÄĘ |vim.version.range()| doesn't exclude `to` if it is equal to `from`.
‚ÄĘ |$VIM| and |$VIMRUNTIME| no longer check for Vim version-specific runtime
  directory `vim{number}` (e.g. `vim82`).
‚ÄĘ 'scrollback' maximum value increased from 100000 to 1000000
